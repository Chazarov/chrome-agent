# Продукт 
    Разработаны инструменты предоставляющие агенту максимально полный доступ к браузерам и к текущей странице.
    
## Информация о странице для агента.

Информация извлекается Parser объектом который хранит состояние. Текущую страницу и позволяет совершать какие либо действия с ними. 

1)   схема Pydantic хранящая всю необходимую информацию о странице для совершения дальнейших действий агентом. Эта модель должна содержать:  
    1) buttons List[Button] - Список доступных для нажатия элементов страницы.  Извлечение этих элементов из страницы должно быть реализовано  в отдельной функции
    2) links - Передать список ссылок на странице. Ссылка - тоже отдельная схема , содержащая поля текст и url
    3) title - золовок страницы.
    4) visible_text - текст страницы 
2)  схема Button -
    1) text - текст кнопки
    2) position - позиция кнопки
    3) parent_text - текст родительского контейнера(или двух) 
    4) Объект Button представляет собой любой блок на странице удовлетворяющий условию:
        - Тег <button> (любой button)
        - ARIA роль [role="button"]
        - Input кнопки input[type="submit"], input[type="button"]
3) схема Link
    1) text
    2) position
    3) url
    4) parent_text

# Требования
- наличие типизации через pydantic. 
- Все схемы находятся в папке models. Каждая схема находится в отдельном скрипте исключение составляют только расширения одной и той же схемы. Например UserBase и UserAdd , UserUpdate модели будут в одном файле , но модель CarScheme будет в отдельном скрипте. 
    - Комментарии: Для каждой модели в начале файла должно быть четко расписано в коментарии для чего она нужна и где используется .
- Не оставлять комментарии кроме явно описаных далее случаев. В таких случаях комментарии оставляются но строго на английском языке.
