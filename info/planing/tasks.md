### 4. little_bind_agent
**Коммит:**
```75718ad070dd6b8579c62a2f7c668291533bd972```
**Детали:**
- Добавлен отладочный модуль `debug_collector` для сохранения и анализа полной информации, предоставляемой агенту системой. Модуль обеспечивает глубокое логирование и форматирование информации запроса, ответа агента, размышлений агента и результатов вызова инструментов.
- Переработан инструмент `get_page_info`: первая реализация бездумно обрезала переданную информацию и конвертировала её в сомнительный текстовый формат. Теперь агент получает JSON с данными о странице напрямую, что значительно улучшило понимание контекста.

### 5. too_large_get_page_info_context
**Коммит::** 
```5b54d79c638939c4fa05c4d2caaa882babdfa5e1```
**Решение:**
- Реализована система пагинации для получения информации о странице: get_page_info заменён на три инструмента — get_page_text_next_item, get_page_buttons_next_item, get_page_links_next_item. Это решает проблему переполнения контекста LLM при работе с большими страницами.
- Реализовано ID-based взаимодействие с элементами страницы. Теперь click_button, type_text, fill_input принимают уникальный ID элемента вместо CSS-селектора. Playwright Locator хранится в BrowserManager, агенту передаётся только информация для принятия решений.
- Добавлен инструмент press_key для нажатия клавиш (Enter, Tab, Escape и др.), что позволяет отправлять формы и выполнять поиск.
